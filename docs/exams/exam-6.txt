1. Question
Category: SOA-C03 – Deployment, Provisioning, and Automation
A commercial bank has recently adopted a hybrid cloud architecture which prompted them to migrate most of their applications to AWS. You are creating multiple CloudFormation stacks for all of the applications that will be used by different departments. You are tasked to ensure that all RDS database instances that you deploy should be kept even if the stack is deleted.

How can you satisfy this requirement?


Set the DeletionPolicy attribute for the stack resource to Retain

Set the DeletionPolicy attribute for the stack resource to false

Set the DeletionPolicy attribute for the stack resource to Snapshot
Enable termination protection of the stack
Incorrect
With the DeletionPolicy attribute, you can preserve or (in some cases) backup a resource when its stack is deleted. You specify a DeletionPolicy attribute for each resource that you want to control. If a resource has no DeletionPolicy attribute, AWS CloudFormation deletes the resource by default.



DeletionPolicy Options:

– Delete: The AWS CloudFormation service deletes the resource and all its content if applicable during stack deletion. You can add this deletion policy to any resource type.

– Retain: The AWS CloudFormation service keeps the resource without deleting the resource or its contents when its stack is deleted.

– Snapshot: The AWS CloudFormation service creates a snapshot for the resource before deleting it.

Hence, the correct answer is: Set the DeletionPolicy attribute for the stack resource to Retain.

The option that says: Set the DeletionPolicy attribute for the stack resource to Snapshot is incorrect because this will just create a final snapshot of the RDS instance before deleting it, and it will not retain the RDS instance after the stack is deleted.

The option that says: Set the DeletionPolicy attribute for the stack resource to false is incorrect because the only valid values for DeletionPolicy are Delete, Retain, and Snapshot.

The option that says: Enable termination protection of the stack is incorrect because it will just prevent the stack from being deleted, but it will not retain the RDS instances after the stack is deleted.

2. Question
Category: SOA-C03 – Reliability and Business Continuity
A leading commercial bank is currently implementing a hybrid cloud architecture. There is a requirement to use Amazon S3 as your primary data storage while retaining frequently accessed data locally in your storage gateway. The objective is to minimize the need to scale your on-premises storage infrastructure while still providing your applications with low-latency access to their frequently accessed data.

Which type of AWS Storage Gateway is best to use for this scenario?

File Gateway
Virtual Tape Library (VTL)
Stored Volume Gateway
Cached Volume Gateway
Incorrect
The Volume Gateway is a cloud-based iSCSI block storage volume to your on-premises applications. The Volume Gateway provides either a local cache or full volumes on-premises while also storing full copies of your volumes in the AWS cloud.



There are two options for Volume Gateway:

Cached Volumes – you store volume data in AWS, with a small portion of recently accessed data in the cache on-premises.

Stored Volumes – you store the entire set of volume data on-premises and store periodic point-in-time backups (snapshots) in AWS.

By using cached volumes, you can use Amazon S3 as your primary data storage while retaining frequently accessed data locally in your storage gateway. Cached volumes minimize the need to scale your on-premises storage infrastructure while still providing your applications with low-latency access to their frequently accessed data. You can create storage volumes up to 32 TiB in size and attach to them as iSCSI devices from your on-premises application servers. Your gateway stores data that you write to these volumes in Amazon S3 and retains recently read data in your on-premises storage gateway’s cache and upload buffer storage.

Hence, the correct answer is: Cached Volume Gateway.

File Gateway is incorrect. Although this type of gateway can also provide low latency access to the recently accessed data via its local cache, it doesn’t meet the requirement of the scenario. Take note that you must use Amazon S3 as your primary data storage while retaining frequently accessed data locally in your storage gateway. In this type, the software appliance or gateway is deployed into your on-premises environment as a virtual machine (VM) running on VMware ESXi, Microsoft Hyper-V, or Linux Kernel-based Virtual Machine (KVM) hypervisor. The gateway only provides access to objects in S3 as files or file share mount points.

Stored Volume Gateway is incorrect because this is only suitable if you need low-latency access to your entire dataset and not just the frequently accessed data.

Virtual Tape Library (VTL) is incorrect because VTL is just a component of Tape Gateway that is like a physical tape library available on-premises with robotic arms and tape drives. This is primarily used for your existing tape-based backup infrastructure to store data on virtual tape cartridges that you create on your tape gateway.

3. Question
Category: SOA-C03 – Deployment, Provisioning, and Automation
A tech company is moving their on-premises database to AWS but still undecided if they want to use RDS or just an EC2 instance. As their Systems Administrator, you are responsible to know the database-related activities that will be performed by AWS, which would eventually lessen the administrative activities of their IT Operations team.

Which of the following is carried out by AWS for database hosted in the AWS RDS service? (Select TWO.)

Query Optimization.
Create and maintain backups with a long term retention of 1 year.
Automatically deploy RDS Read Replicas when the database utilization is high.
Create and maintain backups with a point in time recovery of 5 minutes.
Critical security patches installation.
Incorrect
Periodically, Amazon RDS performs maintenance on Amazon RDS resources. Maintenance most often involves updates to the DB instance’s underlying operating system (OS) or database engine version. Updates to the operating system most often occur for security issues and should be done as soon as possible.



You can restore a DB instance to a specific point in time, creating a new DB instance. RDS uploads transaction logs for DB instances to Amazon S3 every 5 minutes.



Hence, the correct answers are:

– Create and maintain backups with a point in time recovery of 5 minutes.

– Critical security patches installation.

The option that says: Create and maintain backups with a long term retention of 1 year is incorrect because the maximum retention period possible for RDS databases is only 35 days and not 1 year.

The option that says: Query Optimization is incorrect because the customer is responsible for applying query optimizations for the database to improve its performance.

The option that says: Automatically deploy RDS Read Replicas when the database utilization is high is incorrect because RDS does not automatically create Read Replicas nor set up a Multi-AZ deployments configuration.

4. Question
Category: SOA-C03 – Monitoring, Logging, Analysis, Remediation, and Performance Optimization
A company has a newly launched e-commerce website with a Memcached cluster hosted in Amazon ElastiCache. Due to the growth of the site’s user base, the CloudOps Engineer needs to vertically scale the ElastiCache cluster by using a larger instance type.

Which combination of steps should the Engineer implement in this scenario? (Select TWO.)


Specify the new Amazon EC2 instance type in the CacheNodeType parameter.

Use the ModifyCacheCluster API action to modify the instance type of the cluster.

Specify the new Amazon EC2 instance type in the InstanceType parameter.

Use the ModifyCacheParameterGroup API action to create a new ElastiCache cluster.

Use the CreateCacheCluster API action to create a new ElastiCache cluster.
Incorrect
The amount of data your application needs to process is seldom static. It increases and decreases as your business grows or experiences normal fluctuations in demand. If you self-manage your cache, you need to provide sufficient hardware for your demand peaks, which can be expensive. By using Amazon ElastiCache, you can scale to meet current demand, paying only for what you use. ElastiCache enables you to scale your cache to match demand.

Basically, we have 2 types of scaling in Amazon ElastiCache for the Memcached cluster:

  1. Scaling HORIZONTALLY:
1.1. Scaling OUT (Adding Nodes to a Cluster)
1.2. Scaling IN (Removing Nodes from a Cluster)

  2. Scaling VERTICALLY
2.1  Upgrading the node type (by creating a new cluster and using a higher EC2 type)
2.1  Downgrading the node type (by creating a new cluster and using a lower EC2 type)

Horizontal and Vertical Scaling

When you scale your Memcached cluster up or down, you must create a new cluster. Memcached clusters always start out empty unless your application populates it. Take note that this is only applicable to the Memcached engine. For the Redis engine, you can directly change the instance type of your cluster without creating a new one.

The CreateCacheCluster API creates a new cluster. All nodes in the cluster run the same protocol-compliant cache engine software, either Memcached or Redis. You can set its CacheNodeType parameter to choose the underlying EC2 instance type of the cluster.

Hence, the correct answers are:

– Use the CreateCacheCluster API action to create a new ElastiCache cluster.

– Specify the new Amazon EC2 instance type in the CacheNodeType parameter

The option that says: Use the ModifyCacheCluster API action to modify the instance type of the cluster is incorrect because, unlike Redis, you cannot directly upgrade the underlying instance type (node type) of your Memcached cluster in ElastiCache. The option that says: Use the ModifyCacheParameterGroup API action to create a new ElastiCache cluster is incorrect because this API simply modifies the parameters of a cache parameter group.

The option that says: Specify the new Amazon EC2 instance type in the InstanceType parameter is incorrect because there is no InstanceType parameter in the CreateCacheCluster API. You have to typically use the CacheNodeType parameter instead.

Event pattern

6. Question
Category: SOA-C03 – Security and Compliance
A company has an enterprise web application that uses Amazon RDS PostgreSQL as its database. The Audit team requires that any sensitive data stored in the database be reviewed. For this reason, the company’s CloudOps Engineer must present a list of findings to the team.

Macie automates the discovery of sensitive data, such as personally identifiable information (PII) and financial data, to provide you with a better understanding of the data that your organization stores in Amazon S3.

If Macie detects sensitive data or potential issues with the security or privacy of your data, it creates detailed findings for you to review and remediate as necessary. You can review and analyze these findings directly in Macie, or monitor and process them by using other services, applications, and systems.

Category: SOA-C03 – Deployment, Provisioning, and Automation

Cold HDD
Provisioned IOPS SSD
General Purpose SSD

Incorrect

A broad range of workloads, such as relational and non-relational databases, enterprise applications, containerized applications, big data analytics engines, file systems, and media workflows, are widely deployed on Amazon EBS.

Access policies you attach to your resources (buckets and objects) are referred to as resource-based policies. You may choose to use resource-based policies, user policies, or some combination of these to manage permissions to your Amazon S3 resources.

Hence, the correct answer is: Attach an S3 bucket policy.

The option that says: Enable Versioning on the S3 bucket is incorrect because this does not prevent the users from deleting objects.

The option that says: Enable MFA Delete on the bucket is incorrect because this is primarily used to prevent accidental deletion of objects from the bucket. Even though you have this feature enabled, users can still delete objects as long as they have a valid MFA device and have been authenticated properly.

The option that says: Enable S3 Access Points is incorrect because access points are simply used for managing access for shared datasets, but they don’t inherently restrict delete operations.
